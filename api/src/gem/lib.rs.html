4//! Loom — GPU-accelerated RTL logic simulator.
5//!
6//! Loom compiles gate-level netlists into GPU-executable Boolean processor
7//! programs and simulates them on CUDA or Metal GPUs, achieving 5–40× speedup
8//! over CPU-based RTL simulators.
9//!
10//! # Pipeline
11//!
12//! ```text
13//! Verilog netlist
14//!   → NetlistDB        (netlistdb crate — flat gate-level database)
15//!   → AIG              (aig — and-inverter graph with DFFs, SRAMs, clock gates)
16//!   → StagedAIG        (staging — pipeline stages for deep circuits)
17//!   → Partitions       (repcut + pe — hypergraph partitioning → GPU block mapping)
18//!   → FlattenedScript   (flatten — packed GPU instruction stream)
19//!   → GPU kernel        (CUDA or Metal — Boolean processor execution)
20//! ```
21//!
22//! # Key modules
23//!
24//! - [`aigpdk`] — AIGPDK standard cell library interface (AND gates, DFFs, clock gates, SRAMs)
25//! - [`sky130`] / [`sky130_pdk`] — SKY130 cell library support with Liberty timing
26//! - [`aig`] — And-inverter graph construction from [`netlistdb::NetlistDB`]
27//! - [`liberty_parser`] — Liberty (.lib) file parser for cell timing data
28//! - [`staging`] — Splits the AIG into pipeline stages via `--level-split` thresholds
29//! - [`repcut`] — Hypergraph partitioning using mt-kahypar
30//! - [`pe`] — Partition executor: maps partitions to GPU block resources (Boomerang stages)
31//! - [`flatten`] — Generates the final packed GPU execution script ([`flatten::FlattenedScriptV1`])
32//! - [`event_buffer`] — Timing event buffer for arrival-time propagation
33//! - [`sdf_parser`] — SDF (Standard Delay Format) file parser for back-annotation
34//! - [`testbench`] — Testbench configuration and VCD-driven simulation setup
35//! - [`display`] — Display/assertion support infrastructure
